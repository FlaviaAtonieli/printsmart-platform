# 🧩 OPS-38 — Definir Stack Tecnológica

## 🎯 Objetivo

Definir oficialmente a stack tecnológica que será utilizada na implementação do PrintSmart, considerando:

- Escopo individual
- Complexidade do motor de regras
- Necessidade de testes automatizados
- Defesa acadêmica
- Produtividade e viabilidade técnica

---

# 🧠 Decisão Oficial

## 🔷 Backend

**Node.js com TypeScript utilizando Next.js (API Routes / Route Handlers)**

Justificativa:

- Ambiente fullstack unificado
- Facilidade de desenvolvimento
- Estrutura modular organizada
- Forte ecossistema REST
- Excelente integração com PostgreSQL
- Facilidade para testes com Jest
- Deploy simplificado na Vercel

---

## 🔷 Arquitetura

**Modular Monolith**

Justificativa:

- Separação clara por domínios
- Baixo acoplamento interno
- Baixa complexidade operacional
- Facilidade de manutenção
- Escalável internamente
- Estrutura defensável academicamente

---

## 🔷 Banco de Dados

**PostgreSQL**

Justificativa:

- Banco relacional robusto
- Suporte completo a modelagem normalizada
- Excelente para regras parametrizadas
- Amplamente aceito no meio acadêmico

---

## 🔷 ORM

**Prisma**

Justificativa:

- Tipagem forte com TypeScript
- Migrations organizadas
- Facilidade de modelagem relacional
- Integração nativa com Node.js

---

## 🔷 Testes

**Jest**

- Suporte a TDD
- Testes unitários simples
- Cobertura automática
- Integração com TypeScript

Meta mínima:

> 60% de cobertura, com foco prioritário no Motor de Regras.

---

## 🔷 Comunicação

- REST API
- JSON
- Stateless

---

# 📦 Stack Final Consolidada

```
Backend: Next.js (Node.js + TypeScript)
Arquitetura: Modular Monolith
Banco: PostgreSQL
ORM: Prisma
Testes: Jest
Comunicação: REST
Deploy: Vercel
```
